VANILLA JAVASCRIPT vs VUE.JS

Mobile apps and desktop apps usually feel very reactive. Things happen instantly and you don't have to wait for pages to load or actions to complete.

In traditional web apps or web pages, you click on a link and waitfor a new page to load. You click on a button and waitfor some action to complete.

JavaScript can help us here to create a more modern app and a more modern user interface.

VANILLA JS

JavaScript has the advantage of running in the browser. Running on the already loaded page without fetching new HTML.

You can build any type of app using Vanilla JavaScript without using a framework like Vue, but:

  • you have to write all the code on your own
  • you might write suboptimal code or introduce errors and bugs
  • working in a team might be harder because not everyone knows how you structure your code

VUE.JS

Such disadvantages of using Vanilla JS to build apps is why we use a framework like Vue.

Vue has the following advantages over Vanilla JavaScript:

  • it gives us a clear ruleset to work with
  • a ruleset that everyone in a team has to use
  • we don't need to figure out in depth details such as updating a page. Instead, you have room to focus on core business logic with Vue and let Vue do the heavy lifting of updating the page

THE DIFFERENT WAYS OF USING VUE

You can use Vue to control parts of HTML pages to make those parts more reactive and interactive.

Vue can also be used to control the entire page. The so-called single page application approach.

LET'S BREAK DOWN THE APP

Here's a look at the HTML code.

  • This div with an id of app is where we connect Vue to our page
  • In the form, we listen for the submit event with the v-on directive
  • We then prevent the default behaviour of the form with the .prevent modifier
  • In the input element, we listen for an input event and at the same time we're writing the value back to the input element using the v-model directive
  • In the p tags, we use string interpolation (double curly braces {{ }}) to reference the values of the properties returned by the data( ) function in JavaScript
  • In the remaining two buttons, we listen for click events with the v-on directive and execute two functions, add(1) and reduce(2). The values passed into the functions are used to increment and decrement the counter

LET'S BREAK DOWN THE APP SOME MORE

Here's a look at the JavaScript code.

  • We create a Vue app with Vue.createApp({ }); the createApp( ) method takes in a JavaScript object where we configure the app
  • The data( ) function always returns an object and it is here that we define the data that our Vue object should be aware of
  • The computed properties are like methods, but Vue will be aware of their dependencies and only re-execute them if one of the dependencies changes. (Methods are re-executed whenever anything on the Vue controlled part of the page changes, hence for performance, it's better to use computed properties.)
  • In the methods options, we define functions or methods that should be called inside the HTML code
  • Calling .mount( ) on our constant variable app let's Vue know which part of the page should be controlled by the Vue app. We pass in a string with a CSS selector. In this case, that's the id of that part of the page.
